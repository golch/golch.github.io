Spring 프레임워크에 대한 내용이 대부분인데, 이 내용을 온전히 이해하려면 Java 기초에 대한 지식이 필요하다. OOP라든지 오버로딩 오버라이딩, 다형성 등의 용어가 익숙하다면 괜찮겠다.

처리 영역
프레임워크
설명
Presentation
Struts
 Struts 프레임워크는 UI Layer에 중점을 두고 개발된 MVC(Model-View-Controller)프레임워크다.
이미 전세계적으로 가장 많이 사용되고 있는 프레임워크이며, 오픈소스 사용을 꺼리고 있는 대부분의 국내 대형 SI 업체들도 Struts를 사용하고 있다.
Spring MVC
 Struts와 동일하게 MVC아키텍쳐를 제공하는 UI Layer 프레임워크다. Spring 프레임워크를 구성하는 작은 모듈이다.
Business
Spring
 Spring 프레임워크는 컨테이너 성격을 가지는 프레임워크다. Spring은 아키텍쳐상에서 비즈니스 Layer에서부터 Persistence Layer, UI Layer 전체적인 영역을 모두 포함하고 있다.
물론 Spring이 비즈니스 Layer만을 전담하고, Persistence Layer, UI Layer는 다른 프레임워크를 사용하여 개발할 수도 있다.
Persistence
Hibernate
 Hibernate 프레임워크는 Persistence Layer를 담당하는 ORM(Object Relation Mapping) 툴이다. 근래에 들어 ORM에 대한 관심도가 높아지면서 Hibernate 프레임워크에 대하여 많은 개발자들이 관심을 가지고 있다.
iBatis
 Ibatis 프레임워크는 SQL Mapper로서 Hibernate와는 달리 완벽한 ORM 도구가 아니다. 개발자가 작성한 SQL 명령어와 Object를 매핑해주는 기능을 제공하며, 기존의 SQL 명령어를 재사용하여 개발하는 애플리케이션에 유용하게 적용할 수 있다.

위 표를 보고 뭔가 느껴지는 것이 있는가?
그렇다. 많이 쓰이는 오픈소스 자바 프레임워크들은 어느 한 가지만으로 구성하는 것이 아니고 서로 조합해서 시너지 효과를 얻을 수 있다.

** 프레임워크 적용 시 고려사항

1. 만들고자 하는 어플리케이션의 성격을 파악한다.

2. 개발하려는 어플리케이션의 아키텍쳐를 명확히 정의한다.

3. 프로젝트의 구성원의 성향을 고려하여 프레임워크를 선택한다.

4. 프레임워크를 적절히 조합한다.

교재의 첫 장이라서 다소 이론적인 이야기들이 나오고 있지만, 추후 이어지는 포스팅에서 실제 연계방법 등도 다루려고 하니 걱정하지 말기를 바란다. 어디까지나 이 포스팅의 목표는 교재 전부를 복습하고 정리하는 것이다.(다른 사람들과 공유하는 것까지..)

* 스프링 프레임워크란?
1. 경량(Lightweight)
2. 제어 역행(IoC - Inversion of Control)
3. 관점지향(AOP - Aspect-Oriented Programming)
4. 컨테이너(Container)
5. Framework

IoC와 AOP를 지원하는 경량의 컨테이너 프레임워크 이다.
