* IoC(Inversion of Control) 컨테이너

IoC(역제어) 란 무엇인가?
객체지향 언어인 자바에서 자바 객체를 생성하고 객체들 사이의 의존 관계를 연결시키는 작업에 대한 제어권은 개발되는 어플리케이션(내가 짜는 코드)에 있다. 이를 임의로 순제어라고 부른다.

반면에 스프링을 이용하면 IoC컨테이너가 객체의 생성과 의존관계에 대한 설정을 해준다. 이를 제어의 역행, 간단히 역제어 라고 한다.


이 IoC는 컴포넌트의 재사용을 용이하게 해주고, 단위테스트를 쉽게 할 수 있도록 지원한다. Spring 프레임워크 내부에서 Spring core 모듈이 바로 이 IoC 컨테이너를 포함하고 있는, 가장 중요한 모듈이고 추후 배울 예제에서 BeanFactory 가 바로 Spring IoC 컨테이너 이다.

이번에는 실습을 통해서 결합도와 유지보수성이 무엇인지 알아보자.

실습을 위해서는 JDK와  Eclipse가 설치되어 있어야 한다. 검색해보면 설치방법을 쉽게 찾을 수 있으니 굳이 언급하지는 않겠다.


자 그럼 아래 예제를 보자.


SamsungTV.java

public class SamsungTV {
public void powerOn(){
System.out.println("SamsungTV --- 전원을 켠다.");
}
public void powerOff(){
System.out.println("SamsungTV --- 전원을 끈다.");
}
public void volumeUp(){
System.out.println("SamsungTV --- 소리를 높인다.");
}
public void volumeDown(){
System.out.println("SamsungTV --- 소리를 낮춘다.");
}
}

LgTV.java

public class LgTV {
public void turnOn(){
System.out.println("LgTV --- 전원을 켠다.");
}
public void turnOff(){
System.out.println("LgTV --- 전원을 끈다.");
}
public void soundUp(){
System.out.println("LgTV --- 소리를 높인다.");
}
public void soundDown(){
System.out.println("LgTV --- 소리를 낮춘다.");
}
}

TVUser.java

public class TVUser {
public static void main(String[] args){
SamsungTV tv = new SamsungTV();
tv.powerOn();
tv.volumeUp();
tv.volumeDown();
tv.powerOff();
}
}

TVUser는 SamsungTV와 LgTV객체를 이용하여 로직을 처리한다. 그런데 이렇게 객체들이 직접 결합되어 있을 경우, TV객체를 다른 객체로 변화하고자 할 때 TVUser 쪽에서 많은 코드를 수정해야 한다.


예를들어, 현재 사용중인 SamsungTV를 LgTV로 바꾸려면 TVUser 에서는 아래와 같은 수정이 필요하다.

수정된 TVUser.java
public class TVUser {
public static void main(String[] args){
// SamsungTV tv = new SamsungTV();
// tv.powerOn();
// tv.volumeUp();
// tv.volumeDown();
// tv.powerOff();
LgTV tv = new LgTV();
tv.turnOn();
tv.soundUp();
tv.soundDown();
tv.turnOff();
}
}


결론은 이런 많은 수정이 불편하다는 것이다. 이렇게 객체를 변경하고자 할 때 많은 부분에서 수정이 발생한다면 유지보수는 어려워지며, 결국 TV객체와 TVUser 사이의 결합도를 떨어뜨려야 기본적으로 유지보수가 용이해진다.


그럼 이번엔 결합도를 떨어뜨리는 방법을 알아보자. 결합도를 낮추기 위한 다양한 방법이 있지만 가장 쉬운 방법은 객체지향의 기본 개념인 다형성(Polymorphism)을 이용하는 것이다. 먼저 TV 클래스들에 대한 인터페이스를 정의하고 SamsungTV와 LgTV를 각각 TV 인터페이스를 구현하도록 한다.




아래 예제코드를 보자.

TV.java
public interface TV {
public void powerOn();
public void powerOff();
public void volumeUp();
public void volumeDown();
}

TV 인터페이스에는 모든 TV 클래스에서 공통으로 가지고 있어야 할 메소드를 선언한다.

SamsungTV.java
public class SamsungTV implements TV{
public void powerOn() {
System.out.println("SamsungTV --- 전원을 켠다.");
}
public void powerOff() {
System.out.println("SamsungTV --- 전원을 끈다.");
}
public void volumeUp() {
System.out.println("SamsungTV --- 소리를 높인.");
}
public void volumeDown() {
System.out.println("SamsungTV --- 소리를 낮춘다.");
}

}


LgTV.java
public class LgTV implements TV{
public void powerOn() {
System.out.println("LgTV --- 전원을 켠다.");
}
public void powerOff() {
System.out.println("LgTV --- 전원을 끈다.");
}
public void volumeUp() {
System.out.println("LgTV --- 소리를 높인.");
}
public void volumeDown() {
System.out.println("LgTV --- 소리를 낮춘다.");
}
}

SamsungTV와 LgTV는 TV 인터페이스를 implements하여 구현한다.

이제 이 두 객체를 이용해서 TVUser클래스를 만들고 수정해보자.

TVUser.java
public class TVUser {
public static void main(String[] args){
TV tv = new SamsungTV();
//TV tv = new LgTV();
tv.powerOn();
tv.volumeUp();
tv.volumeDown();
tv.powerOff();
}
}

이렇게 형변환을 통해서 객체를 참조하게 되면 변경을 원할 때 최소한의 수정으로 원하는 객체를 사용할 수 있어서 유지보수성이 좋아졌다고 볼 수 있다.

  • 다형성 = 상속 + Override + 형변환

이렇게 다형성을 이용한 결합도 낮추기에 대하여 알아보았다.
이번에는 Factory 패턴을 이용한 결합도 낮추기에 대해서 알아보자.
다형성을 이용하면 여러 클래스를 동일 타입으로 이용하여 결합도를 낮출 수 있지만, 그 객체를 이용하기 위해서 생성할 때 발생하는 결합(생성결합)까지 없어지지는 않는다.
아래와 같이 BeanFactory 기능의 클래스를 구현한다.
BeanFactory.java
public class BeanFactory {
    public Object getBean(String beanName){
if(beanName.equals("samsung")){ return new SamsungTV(); }else if(beanName.equals("lg")){
return new LgTV();
} return null; } }

그리고 이 BeanFactory 클래스를 이용하여 TVUser 클래스를 다음과 같이 수정한다.
TVUser.java
public class TVUser {
    public static void main(String[] args){
//TV tv = new SamsungTV(); //TV tv = new LgTV(); BeanFactory factory = new BeanFactory();
TV tv = (TV)factory.getBean("lg");
tv.powerOn();
tv.volumeUp();
tv.volumeDown();
tv.powerOff();
}
}

위 클래스에서는 BeanFactory 클래스를 이용하여 자신이 사용하고자 하는 객체를 요청하고 BeanFactory가 리턴한 객체를 이용하여 적절히 로직을 수행하고 있다. 이렇게 BeanFactory를 이용하면 직접적인 객체 생성코드를 작성하지 않기 때문에 생성결합 문제도 자연스럽게 해결된다. 이런 생성결합을 낮추기 위한 방법이 Factory 패턴이고 이 Factory 패턴이 적용된 것이 컨테이너 기능이다.
