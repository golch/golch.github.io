1. 메서드 오버로딩

자바에는 메서드라는 녀석이 있다.
아래 그림과 같이 메서드는 파라미터(매개변수)와 리턴 값을 가진다.

메서드 오버로딩은 매개변수와 깊은 관련이 있다. 리턴 값과는 전혀 연관이 없음을 알아두길 바란다.

위 예시 코드에서 보면 매개변수가 (int a, int b) 두 개임을 알 수 있다.
그런데 나는 똑같은 기능(Add)을 매개변수 3개를 받아서 사용하고 싶다.
보통 이런 경우에는 메서드의 이름을 다르게 만든다.
그러나 같은 기능을 하는 메서드라면 Add 라는 이름으로 매개변수의 갯수, 자료형만 바꿔서 만들 수 있다.
코드는 아래와 같이 될 것이다.

static int Add(int a, int b, int c)
{
     return a + b + c;
}

이렇게 같은 이름의 메서드를 매개변수만 바꿔서 여러 개 만드는 것을 메서드 오버로딩이라고 한다.
어느 Java 책에나 나오는 전형적인 예시로는 println 메서드가 있다.
String, int, double ...... 등 여러 자료형이 있는데 이를 콘솔창에 나타낼 때에는 println 이라는 한 가지 이름의 메서드를 사용한다.
요 녀석이 사실은 오버로딩 된 메서드이다.

2. 오버라이딩

오버라이딩은 상속과 깊은 관계가 있다.
부모 클래스로부터 상속받은 메서드의 내용을 자식 클래스에서 변경하는 것을 오버라이딩이라고 한다.

가령, 위 그림의 코드를 class A 라고 가정하고 그 자식 클래스 B 를 만들어서 오버라이딩을 해보면 이렇게 될 것이다.

class B extends A {

    static int Add(int a, int b)
    {
         String formula;
         formula = "a + b = ";

         int sum;
         sum = a + b;
         return formula + sum;
    }    

    static int Add(int a, int b, int c)
    {         

         String formula;
         formula = "a + b + c = ";

         int sum;
         sum = a + b + c;
         return formula + sum;

    }

    static void main(){
        int num;
        num = Add(10,20);
    }

}

더한 값 앞에 식을 보여주는 메서드로 내용을 추가하였다.
요 녀석이 오버라이딩이다.

3. 오버로딩과 오버라이딩의 차이점

오버로딩 : 기존에 없는 새로운 메서드를 정의하는 것(new) - 코드로 보면 오버로딩 된 메서드들은 파라미터(매개변수)가 다른 것을 알 수 있다.

오버라이딩 : 상속받은 메서드의 내용을 변경하는 것(change, modify) - 코드를 보면 상속관계에 있는 클래스의 메서드이고 파라미터(매개변수)의 차이가 없이 똑같은 것을 알 수 있다.
